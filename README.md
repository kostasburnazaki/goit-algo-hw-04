Завдання 3 — Порівняння алгоритмів сортування

Було реалізовано та протестовано три алгоритми:

Insertion Sort
Merge Sort
Timsort (вбудована функція Python sorted())

Методика тестування

Для оцінки ефективності алгоритмів використовувався модуль timeit.

Тестування проводилось на масивах різного розміру:

100 елементів
1000 елементів
5000 елементів

Для кожного алгоритму використовувалась копія масиву, щоб забезпечити однакові умови.

Результати (узагальнено)


Insertion Sort
Працює добре лише на малих масивах
При збільшенні розміру різко зростає час виконання
Має складність O(n²)

Merge Sort
Показує стабільний час роботи
Ефективний для великих масивів
Має складність O(n log n)

Timsort (sorted)
Найшвидший у більшості випадків
Оптимізований для реальних даних
Використовує комбінацію Merge Sort та Insertion Sort
Має складність O(n log n), але працює швидше на практиці


Висновки
Insertion Sort є неефективним для великих обсягів даних через квадратичну складність.
Merge Sort значно ефективніший і демонструє стабільну продуктивність незалежно від вхідних даних.
Timsort є найкращим варіантом, оскільки
комбінує переваги Insertion Sort і Merge Sort
використовує частково відсортовані підмасиви (runs)
мінімізує кількість операцій
